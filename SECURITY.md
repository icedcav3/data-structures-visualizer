# Security Policy

## Сообщение об уязвимостях

Используйте GitHub's private vulnerability reporting для безопасного сообщения об уязвимостях.

### Как сообщить

1. Перейдите на вкладку "Security" в репозитории
2. Нажмите "Report a vulnerability"
3. Заполните форму с деталями уязвимости
4. НЕ открывайте публичное Issue

## Версии, получающие обновления

- **1.x** - Активная разработка и патчи безопасности

## Процесс

1. Получение отчета
2. Подтверждение и оценка
3. Разработка патча
4. Тестирование
5. Выпуск и публикация

## Известные проблемы

См. Issues на GitHub для известных проблем.
