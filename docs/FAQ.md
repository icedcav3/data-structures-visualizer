# Часто задаваемые вопросы

## Установка и запуск

### Q: Как установить приложение?
**A:** Смотрите [документацию по развертыванию](DEPLOYMENT.md).

### Q: Какая версия Python требуется?
**A:** Python 3.8 или выше.

### Q: Можно ли запустить приложение без графического интерфейса?
**A:** Нет, приложение требует GUI. Однако можно использовать структуры данных программно.

### Q: Как запустить тесты?
**A:** Используйте `pytest` или `python scripts/run_tests.py`.

## Функциональность

### Q: Что такое отмена (Undo)?
**A:** Это функция, которая отменяет последнюю операцию, возвращая структуру данных к предыдущему состоянию.

### Q: Есть ли ограничения на размер структур?
**A:** Технически нет, но визуализация может быть неудобной при >100 элементов.

### Q: Поддерживаются ли строки в качестве значений?
**A:** Да, приложение поддерживает как числа, так и строки.

### Q: Как экспортировать результаты?
**A:** В настоящее время можно скопировать из журнала операций или просмотреть логи в `data/logs/`.

## Проблемы

### Q: Приложение не запускается
**A:** 
- Проверьте, что Python установлен: `python --version`
- Проверьте зависимости: `pip install -r requirements.txt`
- Проверьте наличие tkinter (см. DEPLOYMENT.md)

### Q: GUI выглядит странно
**A:** Это может быть проблемой масштабирования или темой. Попробуйте:
- Измените размер окна
- Перезагрузитесь
- Проверьте параметры дисплея

### Q: Отмена не работает
**A:** Отмена работает только для операций, которые изменяют структуру (push, pop, enqueue, dequeue, insert, delete). Для операций чтения (peek, front, search) отмена не требуется.

### Q: Потеряны мои данные после закрытия приложения
**A:** Структуры данных не сохраняются между сессиями. Логи операций сохраняются в `data/logs/operations.log.json`.

## Производительность

### Q: Почему приложение медленно при большом количестве элементов?
**A:** Визуализация требует отрисовки всех элементов. Для улучшения производительности:
- Используйте меньше элементов
- Закройте другие приложения
- Уменьшите размер окна

### Q: Как очистить логи?
**A:** Удалите файлы в `data/logs/` или используйте кнопку "Clear Log" в приложении.

## Разработка

### Q: Как добавить новую структуру данных?
**A:**
1. Создайте класс в `src/core/`
2. Создайте Drawer в `src/visualization/`
3. Создайте Controller в `src/controllers/`
4. Обновите GUI для включения новой структуры
5. Добавьте тесты

### Q: Как внести вклад?
**A:** См. раздел в ROADMAP.md.

**A:** Весь код находится на GitHub: https://github.com/icedcav3/data-structures-visualizer
