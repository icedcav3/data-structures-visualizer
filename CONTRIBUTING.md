# Contributing to Data Structures Visualizer

## Как внести вклад

Мы приветствуем вклады от сообщества! Вот несколько способов помочь:

### Сообщение об ошибках

1. Проверьте, что ошибка еще не зарегистрирована
2. Откройте новое Issue с описанием:
   - Что вы ожидали
   - Что произошло
   - Шаги для воспроизведения
   - Версия Python и ОС

### Предложение функций

1. Откройте Issue с шаблоном "Feature request"
2. Опишите использование и преимущества
3. Дайте примеры кода, если применимо

### Pull Requests

1. Форкните репозиторий
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Сделайте коммиты с ясными сообщениями
4. Добавьте или обновите тесты
5. Убедитесь, что все тесты проходят
6. Отправьте Pull Request

### Коммит сообщений

Используйте ясные и описательные сообщения:
- `feat: добавить новую функцию`
- `fix: исправить ошибку в стеке`
- `docs: обновить документацию`
- `test: добавить тесты для BST`
- `refactor: переработать визуализацию`

### Кодовый стиль

- Используйте PEP 8
- Добавляйте docstrings ко всем публичным функциям
- Максимальная длина строки: 100 символов
- Используйте type hints где возможно

### Тестирование

- Напишите тесты для нового кода
- Убедитесь, что все тесты проходят: `pytest`
- Проверьте покрытие: `pytest --cov=src`

### Документация

- Обновите README если добавляете новые функции
- Добавьте docstrings и type hints
- Обновите docs/api/API.md если необходимо

## Гайдлайны развития

### Структура

- `src/core/` - основные структуры данных
- `src/visualization/` - визуализация
- `src/gui/` - графический интерфейс
- `src/controllers/` - логика приложения
- `src/persistence/` - сохранение и конфигурация
- `src/utils/` - вспомогательные функции
- `tests/` - тесты

### Добавление новой структуры данных

1. Создайте класс в `src/core/`
2. Реализуйте основные операции
3. Создайте Drawer в `src/visualization/`
4. Создайте Controller в `src/controllers/`
5. Обновите GUI в `src/gui/controls.py`
6. Добавьте тесты

## Лицензия

Внося вклад, вы соглашаетесь с тем, что ваш код будет распространяться под MIT лицензией.

## Вопросы

Для вопросов используйте GitHub Discussions или откройте Issue.
